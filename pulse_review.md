Pulse — the MCP server that gives AI agents a nervous system
Concept
Pulse is an MCP server that exposes a live human body as agent-readable Resources and gates every real-world action behind consent-scoped, audited Tools. A person wears a Pixel Watch, carries an Android phone, and wears Bluetooth earbuds, the data is fed into a nitrostack MCP server. Through it, any AI agent can read the wearer's physiological and conversational state in real time — heart rate, a derived stress signal, the running transcript, session context, live speech metrics — and respond into the physical world by buzzing the watch or whispering a short line to the earbuds. Because every read and every action passes through the MCP layer (on NitroStack), the agent's access is limited, structured, and fully traceable: it can only touch what the wearer has granted, and every interaction is logged.

The flagship demonstration is a real-time nervous-system co-regulation loop. When the wearer's heart rate climbs and stays elevated, the agent perceives the sustained stress, decides — from context, not a fixed threshold — whether and how to intervene, and acts: a haptic nudge to interrupt the spiral, or a short spoken affirmation to guide the wearer back down. The heart-rate curve then visibly falls, with no human in the loop except the one being helped. The same primitives serve anyone building agents that need to understand and respond to a human's real-time state — anxiety and panic support, high-pressure performance (pitches, interviews, presentations), accessibility and overload early-warning, or frontline-worker stress management.
Core features
Vitals as a Resource — live heart rate plus a derived stress signal (sustained-elevation detection), so the agent reads state, not just raw numbers.
Context + transcript as Resources — pre-loaded session context (who the wearer is, what the situation is) alongside a rolling transcript and speech metrics (pace, filler words), so the agent's decisions are contextual rather than threshold-triggered.
Consent-gated Tools — haptic_nudge (buzz the watch) and whisper_coach (play a short line to the earbuds), each scoped to exactly what the wearer approved. 
Copilot Tool: There can also be an on-demand advice tool that can be triggered to give advice based on the conversation context, this tool would be triggered by a tap on the watch
Audit log — every Resource read and Tool call is timestamped and recorded, queryable after the fact.
HR-synced session report — the transcript and heart-rate trace aligned on a single scrubber; click a spike and hear the exact moment that caused it, with per-session and cross-session trends.
Wow factors
The closed biometric loop, live. Heart rate climbs past ~110 on screen → the watch buzzes → a whispered line lands → the HR curve visibly bends downward, captioned "No human intervened." An AI agent perceiving a human's panic and physically acting to calm it — in real time — is the lean-in moment, and it opens the demo.
Interrogate the MCP. Ask the server "why did you buzz me at 0:14?" and the agent answers citing both the heart-rate spike and the transcript moment that triggered it. Every action is explainable and auditable — the trust-and-safety headline, aimed squarely at judges who care about controlled, traceable agent access.
Human vitals as an MCP primitive. The idea in one sentence: an MCP server that lets any AI agent feel your heartbeat — consent-gated and fully auditable. Exposing a living body as a first-class agent Resource, with physical-world actuation behind permissioned Tools, is a genuinely new primitive rather than another chat-based assistant.
Conversation Copilot: You can ask for on-demand advice for high-stakes conversations like business meetings, startup pitches or ask the server to mention points that you forgot to mention, say for example during a public speech. All pulled from the personalized session context that is fed into the phone app and the transcripts
