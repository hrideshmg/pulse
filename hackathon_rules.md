# **1. Event Overview**

The Amrita University MCP Hackathon 2026 is a 24-hour competitive hackathon designed around the Model Context Protocol (MCP): the open standard that enables AI models to connect with real-world tools, data sources, and services. This event brings together students from across disciplines to design, build, and deploy working MCP servers against real industry problems.

With over 4,000 students on campus and a target participation of 1,000, the hackathon is structured around six industry-aligned tracks. Projects are built using the Nitrostack platform, submitted through the dedicated event portal, and showcased to the wider MCP builder community through McpToTheMoon.

| **McpToTheMoon Community** | Community and visibility partner. All final project submissions must also be posted to the McpToTheMoon Discord server and subreddit to give teams public visibility for their work and connect them with the global MCP builder community. |

# **5. Themes & Industry Tracks**

The hackathon is structured around six industry-aligned tracks. Each track represents a domain where Agentic AI and MCP servers can create measurable real-world impact. Teams must select exactly one track at registration. Judging criteria are identical across all tracks.

| **TRACK** | **THEME & FOCUS AREA** | **EXAMPLE MCP SERVER / APP IDEAS** |
| --- | --- | --- |
| **💰 BFSI & FinTech** | Banking, financial services, insurance — fraud detection, compliance automation, personal finance agents | *Loan eligibility assessment agent · Real-time fraud alert MCP server · Regulatory compliance checker (RBI/SEBI rules) · Portfolio rebalancing advisor · Insurance claim triage MCP* |
| **🏭 Manufacturing & Industry 4.0** | Smart factories, supply chain, quality control, predictive maintenance using agentic AI | *Predictive maintenance agent (sensor data → work order) · Defect detection MCP with vision tools · Supply chain disruption monitor · Production scheduling optimiser MCP* |
| **🌐 IoT & Smart Infrastructure** | Connected devices, smart cities, utilities, building management systems | *Smart grid demand-response agent · Building energy management MCP · Fleet tracking & route optimiser · Smart irrigation controller · Anomaly detection for industrial sensors* |
| **🏥 HealthTech & Life Sciences** | Patient care, diagnostics, hospital operations, pharma research | *Clinical trial data aggregator MCP · Drug interaction checker agent · Appointment & bed management MCP · Medical literature summariser · Remote patient monitoring alert agent* |
| **📚 Education & Research** | Learning management, assessment, academic research, campus administration | *Adaptive assessment generator MCP · Research paper discovery & summariser · Campus timetable & resource booking agent · Student academic progress monitor · Plagiarism & citation checker MCP* |
| **🔓 Open Innovation** | Any domain not listed above — legal, agriculture, logistics, governance, creator economy, cyber security | *Legal contract analyser MCP · Crop advisory agent (weather + soil data) · Last-mile logistics optimiser · Cyber threat intelligence MCP · Government scheme eligibility checker* |

| **What makes a strong track entry?**  The best submissions are not generic chatbots. They are purpose-built MCP servers that expose domain-specific tools and data sources. For example, an MCP server that connects an AI agent to live NSE/BSE market data and executes rule-based portfolio checks (BFSI), or one that ingests real-time sensor streams from IoT devices and triggers maintenance workflows (Manufacturing). Focus on agent functionality. |
| --- |

*The Open Innovation track exists for teams whose idea genuinely does not fit the other five tracks. Teams selecting Open Innovation must clearly justify their domain choice in the submission write-up.*

# **6. Judging, Evaluation & Panel**

Projects are evaluated by a panel of three judges: one from each of the three partner organisations. Judges assess submissions independently using the weighted rubric below. Scores are aggregated to identify the top three teams per track.

## **Judging Panel**

| **ORGANISATION** | **JUDGE ROLE** | **FOCUS AREA** |
| --- | --- | --- |
| **Wekan Enterprise Solutions** | **Industry Judge** | Evaluates real-world applicability, enterprise viability, and domain relevance of submissions |
| **Nitrostack** | **Technical Judge** | Evaluates MCP implementation quality, code architecture, deployment completeness, and Nitrostack platform usage |
| **Amrita University** | **Academic Judge** | Evaluates innovation, research potential, learning outcomes, and educational impact of the project |

## **Evaluation Rubric**

| **CRITERION** | **WHAT WE LOOK FOR** | **WEIGHT** |
| --- | --- | --- |
| **Technical Quality** | Clean MCP server implementation: correct use of Tools, Resources & Prompts; well-structured code; error handling; security considerations | **25%** |
| **Innovation & Creativity** | Originality of the idea; novel use of MCP primitives; unexpected or impactful combinations of tools and data sources | **25%** |
| **Real-World Impact** | Does it solve a genuine industry problem? Is the use case compelling, feasible, and scalable in the chosen track domain? | **20%** |
| **Demo & Presentation** | Quality of the 2-minute video; clarity of the deployed link; storytelling, pitch coherence, and communication of value | **15%** |
| **Completeness** | Working deployment on Nitrostack; integration with at least one external data source or industry API; end-to-end functionality | **10%** |
| **Community Visibility** | Quality of the McpToTheMoon Discord and Reddit post — this is for showcasing work, not scored competitively | **5%** |

| **Note on Community Visibility (5%)**  The McpToTheMoon Discord and Reddit post is primarily a platform for showcasing your work to the global MCP builder community and gaining visibility as a developer. The 5% weight recognises the effort of communicating your project clearly to a public audience. It is not a popularity contest and community votes are not counted. |
| --- |

## **Judging Process**

1. 06:01 PM: Automated checks run: deployment reachability, submission completeness, MCP primitive count.
2. 06:00–06:45 PM — Judges independently review demo videos and written descriptions.
3. 06:45–07:00 PM — Scores aggregated and winners confirmed by all three judges.
4. 07:00 PM — Closing Ceremony begins; winners announced.
